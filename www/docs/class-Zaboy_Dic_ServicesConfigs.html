Class Zaboy_Dic_ServicesConfigs

Zaboy_Dic_ServicesConfigs

Services Config
Usialy Services Config is part of DIC config in application.ini with key which is defined in Zaboy_Application_Resource_Dic::CONFIG_KEY_SERVICE
DIC have got config form resurce plugin Zaboy_Application_Resource_Dic
This is part of application.ini:

dic.services.serviceName.class = ServiceClass
dic.services.serviceName.options.key11 = val11
dic.services.serviceName.options.key12 = val12
dic.services.serviceName.autoload = true
dic.services.serviceName.params.secondParam = OtherServiceName
dic.services.nextServiceName.class = NextServiceClass
...

It is in $servicesConfig (seesetConfigsServices())

array(
    'serviceName' = array(
        'class' = 'ServiceClass'
        'options' = array(
            'key1' = val1
            'key2' = val2
        (
        'autoload' = true
        'params' = array(
            'secondParam' = 'OtherServiceName'
        )
    )
)
array(
    'nextServiceName' = array(
        'class' = 'NextServiceClass'
         ...
There are next keys for using after 'dic.services.' in appalication .ini:
  • class See:Zaboy_Services_Interface::CONFIG_KEY_CLASS
  • instance See:Zaboy_Services_Interface::CONFIG_KEY_INSTANCE
    • singleton See:Zaboy_Services_Interface::CONFIG_VALUE_SINGLETON
    • clone See:Zaboy_Services_Interface::CONFIG_VALUE_CLONE
    • recreate See:Zaboy_Services_Interface::CONFIG_VALUE_RECREATE
  • options See:Zaboy_Services_Interface::CONFIG_KEY_OPTIONS
  • params See:Zaboy_Services_Interface::CONFIG_KEY_PARAMS
