 14:  *  <b>Zaboy_Dic</b>

 16:  * Zaboy_Dic is dependency injection container class (<i>Dic</i>). <i>Dic</i> is loading as resurce plugin of Bootstrap. Add in <i>config.ini</i> for load<br>
 17:  * <code> 
 18:  * resources.dic[] = 
 19:  * </code>
 20:  * You can get <i>Dic</i>: <br>
 21:  * <code> 
 22:  *   $bootstrap->bootstrap('dic');
 23:  *   $dic = $bootstrap->getResource('dic');
 24:  * </code>
 25:  * <br>
 26:  * It can load and contane any object which implements {@see Zaboy_Dic_Service_Interface}.
 27:  * That object we call <i>Service</i>. Specific of <i>Service</i> is parametrs of it's __consruct. <br>
 28:  * The <b>$options</b> ( if it exist as parametr for __construct ) have to first parametr. 
 29:  * More about <i>__consruct()</i> - see {@see Zaboy_Services_Abstract::__consruct()}.<br>
 30:  * More about <i>$options</i> - see {@see Zaboy_Abstract::setOptions()}.<br>
 31:  * You can note options for <i>Service</i>  in config.ini <br>
 32:  * <code> 
 33:  * resources.dic.service.NameOfService.options.param1 = value1
 34:  * resources.dic.service.NameOfService.options.param2 = value2
 35:  * </code>
 36:  * or just in SeviceObjectClass - see $_defaultOptions in {@see Zaboy_Services_Abstract}.<br>
 37:  * 
 38:  * <b>All other parametrs must be Services!</b>. That parametr we call <i>Service Param.</i> <br>
 39:  * <i>Service Param</i> in declaration of __construct must has type.<br>
 40:  * <code>
 41:  *     public function __construct($options, Test_Service_First $testFirst, Test_Service_Next $testNext) 
 42:  *   { ...}
 43:  * </code>
 44:  * <br>
 45:  * You can override type for <i>Service</i> in config.ini <br>
 46:  * For example:<br>
 47:  * <code> 
 48:  * resources.dic.service.NameOfService.class = New_Class_For_NameOfService
 49:  * </code>
 50:  * 
 51:  * <b>About optional parametrs</b><br>
 52:  * Optional  parametrs are transmitted to _consruct if they already was loaded or if they is discribed in config.ini
 53:  * 
 54:  * <b>Aliases</b><br>
 55:  * You can take 2 or more instaces of <i>Service</i>. Use aliases.
 56:  * <code>
 57:  *     public function __construct(Service_One_Type $firstExemplar, Service_One_Type $secondExemplar) 
 58:  *   { ...}
 59:  * </code>
 60:  * <br>
 61:  * You can load <i>Service</i> () see {@see Zaboy_Dic::get()}:
 62:  * <code>
 63:  *    $dic->get('NameOrAlias'); //if class isn't noted in cofig.ini - exception
