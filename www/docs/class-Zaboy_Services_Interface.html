# getDefaultServiceConfig( )

Return Default Config on that case if some keys is absent in application.ini

Return Default Config on that case if some keys is absent in application.ini

Function return default configuration of service. If some keys of config aren't defined in application.ini, configuration for these keys from returned array is used. If any key of top level is present in application.ini - this part of Default Config is ignored completely

string CONFIG_KEY_INSTANCE 'instance'
#

You can specified how Service will be loaded by Dic.
By default, all calls

$dic->get('serviceName');

return singleton. Example of use in application.ini:

 dic.services.serviceName1.instance = singleton

string CONFIG_KEY_OPTIONS 'options'
#

If __construct() has parameter $options ( it have to be first ), __construct() will retrived from Dic empty array in this parameter by default. But you can define options array in application.ini:

dic.services.serviceName.options.key1 = val-1
dic.services.serviceName.options.key2[] = val-2_1
dic.services.serviceName.options.key2[] = val1-2_2
dic.services.serviceName.options.key3.arrayKey1 = val-3_1
dic.services.serviceName.options.key3.arrayKey2 = val-3_2

Dic will assign $options:

array(
    'key1' => 'val-1',
    'key2' => array( 'val-2_1', 'val1-2_2'),
    'key3' => array(
        'arrayKey1' => 'val-3_1',
        'arrayKey2' => 'val-3_1'
    )
)

See also Zaboy_Abstract::setOptions()

string CONFIG_KEY_PARAMS 'params'
#

You can instruct the Dic what kaind of Service have to be given for each parameter. Example of use in application.ini:

dic.services.serviceIsHadParm.params.paramName1 = serviceForParam1

dic.services.serviceForParam1.class = One_Class_Name

string CONFIG_KEY_AUTOLOAD 'autoload'
#

You can load some Service automaticly, just after Dic will be loaded. Add to these services descriptions in application.ini:

dic.services.serviceName1.autoload = true

Default value for this key is FALSE. See also Zaboy_Services_Interface::getDefaultServiceConfig()
