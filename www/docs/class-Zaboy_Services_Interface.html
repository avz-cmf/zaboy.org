Interface Zaboy_Services_Interface

Interface Zaboy_Services_Interface - it is Interface for Service Objects classes

There are 4 different entities: Service, Service Name, Service Object, Class of Service Object

Service
Service is resurse (Service Objec) which was loaded and registered in Zaboy_Dic via Zaboy_Dic::get() with Service Name
Service = Service Name + Service Object

Service Object class
Class of Service Object may be is inherited from Zaboy_Services_Abstract or not There are requirements for (@see Zaboy_Services_Abstract::__construct()}:

Object_Class::__construct()
Constructor's params must be:
  • (array $options, Servece_Class $servece1, Another_Servece_Class $servece2, ...) or
  • (Servece_Class $servece1, Another_Servece_Class $servece2, ...)) or
  • (array $options). See Zaboy_Services_Example_Options. Or just
  • public function __construct(). See Zaboy_Services_Example_Simple
$options have to first or absent
You have to call 
        parent::__construct($options);
// or
        parent::__construct();

in your constructor if parent::__construct() exist.

Object_Class::_defaultOptions
If Service Object is inherited from Zaboy_Services_Abstract it contains Zaboy_Services_Abstract::$_defaultOptions
If $options haven't submitted via __construct - _defaultOptions will be used.
See Zaboy_Services_Example_SimpleTest::testSetDefaultOptionsInConstruct()
If $options is not empty - _defaultOptions will not be used fully ( any part)
See Zaboy_Services_Example_OptionsTest::testSetDefaultOptionsInConstruct()

Also, you can describe object and $options for this object in config.ini. See about it (@see Zaboy_Dic_Interface}
